Native UI and Postback

Despite the cross-platform UIs like <p>...</p>, Machinat also support even more platform specific UIs. In this lesson, we will learn how to use them to provide more intuitive and smooth user experiences.

　Time to accomplish: 10 minutes

Native Component#

Most chat platform provide some widget like buttons or other special features in chatroom. In Machinat, they are available as Native Component of each platform.

Let's try adding a button in the meassge to list user's todos. Follow the guide of the platform you choose to edit handleMessage handler:

  • Messenger
  • Telegram
  • LINE
src/handlers/handleMessage.tsx
import Machinat from '@machinat/core';
import * as Messenger from '@machinat/messenger/components';
import { ChatEventContext } from '../types';
const handleMessage = async ({
event,
reply,
}: ChatEventContext & { event: { category: 'message' } } ) => {
if (event.type === 'text') {
const matchingAddTodo = event.text.match(/add(\s+todo)?(.*)/i);
if (matchingAddTodo) {
const todoName = matchingAddTodo[2].trim();
return reply(
<>
<p>Todo "<b>{todoName}</b>" is added!</p>
<Messenger.ButtonTemplate
buttons={
<Messenger.PostbackButton
title="Show Todos 📑"
payload={JSON.stringify({ type: 'list' })}
/>
}
>
You have <b>{3}</b> todos now.
</Messenger.ButtonTemplate>
</>
);
}
}
// ...
  • Messenger.ButtonTemplate sends a text message with buttons.
  • Messenger.PostbackButton add a button to post payload back.

The tag like Messenger.ButtonTemplate is a Native Component of Messenger platform. These components represent platform specific features, and can only be sent to the platform they belongs to.

Props#

The attributes we put on the JSX element are Props of a component. For example:

<Messenger.ButtonTemplate
buttons={
<Messenger.PostbackButton
title="Show Todos 📑"
payload={JSON.stringify({ type: 'list' })}
/>
}
>
You have <b>{3}</b> todos now.
</Messenger.ButtonTemplate>

Like in React.js, they are used to pass values to the component. The value can be text (like title), expression (like payload) or even another JSX element (like buttons).

We pass JSON encoded data on payload (or data) prop to indicate the action type. We can use the data later to handle the postback event.

Event Flow#

Handle Postback Event#

The button we added will post data back to our server when user tap the button in the chatroom. We can handle the postback events by adding these codes:

  • Messenger
  • Telegram
  • LINE

Create a new src/handlers/handlePostback.tsx file with following content:

src/handlers/handlePostback.tsx
import Machinat from '@machinat/core';
import { ChatEventContext } from '../types';
const handlePostback = async ({
event,
reply,
}: ChatEventContext & { event: { type: 'postback' } } ) => {
const action = JSON.parse(event.data);
if (action.type === 'list') {
return reply(
<>
<p>You have <b>3</b> todos:</p>
<p>......</p>
</>
);
}
};
export default handlePostback;

The 'postback' events will contain the JSON data that we have attached with the sent button. We can get the data at event.data, and then reply user according to the content.

Create a new src/handlers/handlePostback.tsx file with following content:

src/handlers/handlePostback.tsx
import Machinat from '@machinat/core';
import { AnswerCallbackQuery } from '@machinat/telegram/components';
import { ChatEventContext } from '../types';
const handlePostback = async ({
event,
reply,
}: ChatEventContext & { event: { type: 'callback_query' } } ) => {
const action = JSON.parse(event.data!);
const answerQuery = <AnswerCallbackQuery queryId={event.queryId} />;
if (action.type === 'list') {
return reply(
<>
{answerQuery}
<p>You have <b>3</b> todos:</p>
<p>......</p>
</>
);
}
return reply(answerQuery);
};
export default handlePostback;

The 'callback_query' events will contain the JSON data that we have attached with the sent button. We can get the data at event.data, and then reply user according to the content.

info

The callback_query event of Telegram need to be replied with AnswerCallbackQuery to stop the client side pending.

Stream Flow in main#

One more thing to do is wiring the handlePostback handler with main event stream. Add these codes in the main function:

  • Messenger
  • Telegram
  • LINE
src/main.tsx
import handlePostback from './handlers/handlePostback';
// ...
const main = (event$: Stream<AppEventContext>): void => {
// ...
event$
.pipe(
filter(
(ctx): ctx is ChatEventContext & { event: { type: 'postback' } } =>
ctx.event.type === 'postback'
)
)
.subscribe(handlePostback);
// ...
src/main.tsx
import handlePostback from './handlers/handlePostback';
// ...
const main = (event$: Stream<AppEventContext>): void => {
// ...
event$
.pipe(
filter(
(ctx): ctx is ChatEventContext & {
event: { type: 'callback_query' };
} => ctx.event.type === 'callback_query'
)
)
.subscribe(handlePostback);
// ...
src/main.tsx
import handlePostback from './handlers/handlePostback';
// ...
const main = (event$: Stream<AppEventContext>): void => {
// ...
event$
.pipe(
filter(
(ctx): ctx is ChatEventContext & { event: { type: 'postback' } } =>
ctx.event.type === 'postback'
)
)
.subscribe(handlePostback);
// ...

Now try tap the Show Todos 📑 button in the chat room and you should see:

The event$ argument of main function is a stream of all events. Every event from the chat platform will flow through it. You can .subscribe() the stream to listen events, or use .pipe() to create a new stream.

event$
.pipe(
filter(({ event }) => {
// return `true` to allow only 'postback' event passing
return event.type === 'postback'
})
)
.subscribe(({ event }) => {
// receive only 'postback' event here
});

We use the filter(...) operator to allow only the postback events to pass. The filtered stream is then handled by the handlePostback handler.
