Service and State

Despite replying messages, a chatbot could need many other services to provide more features. In this lesson, you will learn how to use built-in DI (dependencies injection) system to access chat state and other services.

Consuming Services#

Use Services in Container#

Fetching user name is a common feature you might need to make your bot chat better. Machinat.BaseProfiler can be used to achieve that.

Edit the handleMessage handler like this:

src/handlers/handleMessage.tsx
import Machinat from '@machinat/core';
import { makeContainer } from '@machinat/core/service';
import WithMenu from '../components/WithMenu';
import { ChatEventContext } from '../types';
const handleMessage = makeContainer({
deps: [Machinat.BaseProfiler] as const,
})(
(profiler) =>
async ({
event,
reply,
}: ChatEventContext & { event: { category: 'message' } }) => {
if (event.type === 'text') {
// ...
}
const userProfile = await profiler.getUserProfile(event.user);
await reply(
<WithMenu todoCount={3}>
<p>Hello, {userProfile.name}! I'm a Todo Bot 🤖</p>
</WithMenu>
);
}
);
export default handleMessage;

The makeContainer function declares a Service Container with the required deps services. A container event handler is a factory function returning the original handler. Framework would automatically inject the deps instances into the factory in the runtime.

We register the Machinat.BaseProfiler service in deps array. It help us to fetch user profile across the platforms. The profiler instance is then injected into the factory function, and can be used in the original handler.

Use User State#

The getUserProfile method makes a API call to chat platform. It's better to save the user profile locally for the first time, so we can reuse it later on.

Add these codes to store the profile in the state:

src/handlers/handleMessage.tsx
import Machinat, { MachinatProfile } from '@machinat/core';
import { makeContainer } from '@machinat/core/service';
import StateController from '@machinat/core/base/StateController';
import WithMenu from '../components/WithMenu';
import { ChatEventContext } from '../types';
const handleMessage = makeContainer({
deps: [Machinat.BaseProfiler, StateController] as const,
})(
(profiler, stateController) =>
async ({
event,
reply,
}: ChatEventContext & { event: { category: 'message' } }) => {
if (event.type === 'text') {
// ...
}
const userState = stateController.userState(event.user);
let userProfile = await userState.get<MachinatProfile>('profile_cache');
if (!userProfile) {
userProfile = await profiler.getUserProfile(event.user);
await userState.set('profile_cache', userProfile);
}
await reply(
<WithMenu todoCount={3}>
<p>Hello, {userProfile.name}! I'm a Todo Bot 🤖</p>
</WithMenu>
);
}
);
export default handleMessage;

We add a StateController service in deps to cache the profile data locally. The .userState(event.user) returns a state object to access data of event.user.

The state object works like an asynchronous version of Map. We get the local cache value with 'profile_cache' key. If no cached data, we fetch the user profile data and save it for later use.

Use Channel State#

Now let's store the todos data into state while adding todo. First add the type of the todos state data:

src/types.ts
export type TodoState = {
currentId: number;
todos: Todo[];
finishedTodos: Todo[];
};

Then add the following codes in handleMessage handler:

src/handlers/handleMessage.tsx
import { TodoState } from '../types';
// ...
if (event.type === 'text') {
const matchingAddTodo = event.text.match(/add(\s+todo)?(.*)/i);
if (matchingAddTodo) {
const todoName = matchingAddTodo[2].trim();
const todoData = await stateController
.channelState(event.channel)
.update<TodoState>(
'todo_data',
(currentState = { currentId: 0, todos: [], finishedTodos: [] }) => {
const { currentId, todos, finishedTodos } = currentState;
return {
currentId: currentId + 1,
todos: [...todos, { id: currentId + 1, name }],
finishedTodos,
};
}
);
return reply(
<RootMenu todoCount={todoData.todos.length}>
<p>Todo "<b>{todoName}</b>" is added!</p>
</RootMenu>
);
}
}
// ...

We use .update(key, updateFn) method this time to save todo data. It takes a updator function take transform the current state value to the updated value. As you can see, we add the new todo and increment the id counter on the returning data.

If the current state value is empty, undefined would be passed to the updator function. We can simply use a default parameter like = { currentId: 0, todos: [] } to initiate the state.

Now try adding a todo with add todo <name> command. You should see the src/.state_data.json file has content like:

"channelStates": {
"messenger.1234567890.psid.9876543210": {
"todo_data": {
"currentId": 1,
"todos": [
{
"id": 1,
"name": "foo"
}
]
}
}
},

Different from the profile data, we save the todo data to the channel state. event.channel refers to the chatroom where the message are sent to. The scope of channel state and user state is different because of group chat.

Providing Services#

Make Your Own Service#

In the project we need to access the todo data serveral times. To keep the codes clean and DRY, it's better having a service to manage all the todo operations.

For not repeating similar steps, please download the TodoController.ts file with this command:

curl -o ./src/services/TodoController.ts --create-dirs https://raw.githubusercontent.com/machinat/todo-example/main/src/services/TodoController.ts```

In the file we declare a TodoController class to manage todos from the state. Check the end of src/services/TodoController.ts file, you'll see StateController class is wrapped by makeClassProvider function:

export default makeClassProvider({
deps: [StateController],
})(TodoController);

The makeClassProvider function describe how a class should be provided as a service in the DI system. Like container, the required deps would be injected when a new instance is constructed.

Register Services#

A new service must be registered in the app before using. Add the TodoController service in the services section when createApp, like:

src/app.ts
import TodoController from './services/TodoController';
//...
const app = Machinat.createApp({
services: [
// ...
TodoController,
],
});
// ...

Use TodoController#

Now the TodoController service can be used just like the built-in services. Edit handleMessage function like this:

src/handlers/handleMessage.tsx
import TodoController from '../service/TodoController';
// ...
const handleMessage = makeContainer({
deps: [Machinat.BaseProfiler, StateController, TodoController] as const,
})(
(profiler, stateController, todoController) =>
async ({
event,
reply,
}: ChatEventContext & { event: { category: 'message' } }) => {
if (event.type === 'text') {
const matchingAddTodo = event.text.match(/add(\s+todo)?(.*)/i);
if (matchingAddTodo) {
const todoName = matchingAddTodo[2].trim();
const { data } = await todoController.addTodo(event.channel, todoName);
return reply(
<RootMenu todoCount={data.todos.length}>
<p>Todo "<b>{todoName}</b>" is added!</p>
</RootMenu>
);
}
}
// ...
const { data } = await todoController.getTodos(event.channel);
await reply(
<WithMenu todoCount={data.todos.length}>
<p>Hello, {userProfile.name}! I'm a Todo Bot 🤖</p>
</WithMenu>
);
}
);
export default handleMessage;

And the listing todos and finishing todo features can be easilly completed like this:

  • Telegram
  • LINE
src/handlers/handlePostback.tsx
import Machinat from '@machinat/core';
import { makeContainer } from '@machinat/core/service';
import TodoController from '../services/TodoController';
import TodoList from '../components/TodoList'
import WithMenu from '../components/WithMenu';
import { ChatEventContext } from '../types';
const handlePostback = makeContainer({ deps: [TodoController] })(
(todoController) =>
async ({
event,
reply,
}: ChatEventContext & { event: { type: 'postback' } }) => {
const action = JSON.parse(event.data!);
if (action.type === 'list') {
const { data } = await todoController.getTodos(event.channel);
return reply(
<ShowTodoList todos={data.todos} />
);
}
if (action.type === 'finish') {
const { todo, data } = await todoController.finishTodo(
event.channel,
action.id
);
return reply(
<WithMenu todoCount={data.todos.length}>
{todo ? (
<p>Todo "<b>{todo.name}</b>" is done!</p>
) : (
<p>This todo is closed.</p>
)}
</WithMenu>
);
}
}
);
export default handlePostback;
src/handlers/handlePostback.tsx
import Machinat from '@machinat/core';
import { makeContainer } from '@machinat/core/service';
import { AnswerCallbackQuery } from '@machinat/telegram/components';
import TodoController from '../services/TodoController';
import TodoList from '../components/TodoList'
import WithMenu from '../components/WithMenu';
import { ChatEventContext } from '../types';
const handlePostback = makeContainer({ deps: [TodoController] })(
(todoController) =>
async ({
event,
reply,
}: ChatEventContext & { event: { type: 'callback_query' } }) => {
const action = JSON.parse(event.data);
const answerQuery = <AnswerCallbackQuery queryId={event.queryId} />;
if (action.type === 'list') {
const { data } = await todoController.getTodos(event.channel!);
return reply(
<>
{answerQuery}
<ShowTodoList todos={data.todos} />
</>
);
}
if (action.type === 'finish') {
const { todo, data } = await todoController.finishTodo(
event.channel!,
action.id
);
return reply(
<WithMenu todoCount={data.todos.length}>
{answerQuery}
{todo ? (
<p>Todo "<b>{todo.name}</b>" is done!</p>
) : (
<p>This todo is closed.</p>
)}
</WithMenu>
);
}
return reply(answerQuery);
}
);
export default handlePostback;
