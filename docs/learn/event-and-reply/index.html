Event and Reply

Receive a message and send the answer back is the most basic function of chatbots. In this lesson, you will learn how to receive specific type of event and make the response accordingly.

　Time to accomplish: 15 minutes

Listen to Event#

Handle Text Message#

Text messages are the most common message type your bot would meet. Let's enable your bot to handle an "add todo <todo name>" command.

Edit the src/handlers/handleMessage.tsx handler function like this:

src/handlers/handleMessage.tsx
import Machinat from '@machinat/core';import { ChatEventContext } from '../types';
const handleMessage = async ({  event,  reply,}: ChatEventContext & { event: { category: 'message' } }) => {  if (event.type === 'text') {    const matchingAddTodo = event.text.match(/add(\s+todo)?(.*)/i);
    if (matchingAddTodo) {      const todoName = matchingAddTodo[2].trim();      return reply(<p>Todo "<b>{todoName}</b>" is added!</p>);    }  }
  await reply("Hello! I'm a Todo Bot 🤖");};
export default handleMessage;

Now your bot can accept add todo <name> command like this:

event Object#

The handleMessage handler receive a event context object when user sends a message to your bot. You can get the event infomations from the event property. Here are the common infos in event:

{  platform: 'messenger',  // chat platform  category: 'message',    // event category  type: 'text',           // event type  payload: {/*...*/},     // raw data from chat platform  user: {/*...*/},        // user object  channel: {/*...*/},     // chatroom object}

You can use the platform, category and type combination to identify kinds of events. All message event with 'text' type implement the event.text interface. So we can extract the todo name with regular expression safely.

if (event.type === 'text') {  event.text; // string}

Reply Message#

The reply function from the context sends messages back to the chatroom. The easiest way to use it is replying a string like this:

await reply('hello world');

JSX Syntax#

info

To use the JSX syntax in typescript, you must use .tsx file extension. And also import Machinat at the beginning of the file like this:

import Machinat from '@machinat/core';

The html alike syntax <p>Todo "<b>{todoName}</b>" is added!</p> is called JSX. <p>...</p> element represent a message bubble containing the inner text children.

Despite plain text like Todo, element children can contain other elements like <b>...</b>. An expression can also be inserted with the {...} syntax like {todoName}.

JSX is used to construct messages in Machinat framework. We can simply send a JSX element like this:

  await reply(<p>Hello! I'm a Todo Bot 🤖</p>);

Formatting Text#

info

Effect of text format can be different depends on chat platforms:

  • Messenger only support in the desktop browser. It shows markdown syntax in mobile app.
  • LINE doesn't support text format and shows plain text only.

We use the "<b>{todoName}</b>" element to make the todo name bold. Try change the element tag to i, s or code and see what's different.

These text formatting tags are textual and work as plain text so they can be placed in <p></p> element.

<p><b>Hello!</b> <s>I'm a</s> <i>Todo</i> <code>Bot 🤖</code></p> message looks like this:

General Tags#

The element with uncaptalized tags like p and b are general tags, which are supported on all chat platforms. A JSX message contains only general tag element and text is the easiest way to make a cross-platform reply.

For example this meassge can be replied to any chat platform:

await reply(  <>    <p>This can be sent to <b>ALL</b> chat platforms</p>    <img src="https://machinat.com/thumbs_up.png" />  </>);

You can check the full list of general tags here.
