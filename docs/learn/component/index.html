UI Component

Now we know about how to send messages in JSX. In this lesson we will go deeper to make customized chat UI component.

　Time to accomplish: 10 minutes

Building Component#

Carousel Template#

Let's continue to create the UI for showing todos list. But this time in a reusalbe and component.

First add a type represent the todo data:

src/types.ts
export type Todo = {
id: number;
name: string;
};

Follow the guide of the platform you choose to create your first component:

  • Messenger
  • Telegram
  • LINE

Create a src/components/TodoList.tsx file with following content:

src/components/TodoList.tsx
import Machinat from '@machinat/core';
import * as Messenger from '@machinat/messenger/components';
import { Todo } from '../types';
type TodoListProps = {
todos: Todo[];
};
const TodoList = ({ todos }: TodoListProps, { platform }) => {
if (todos.length === 0) {
return <p>You have no todo now.</p>;
}
const summary = <p>You have <b>{todos.length}</b> todos:</p>;
const finishLabel = 'Done ✓';
if (platform === 'messenger') {
return (
<>
{summary}
<Messenger.GenericTemplate>
{todos.slice(0, 10).map((todo) => (
<Messenger.GenericItem
title={todo.name}
buttons={
<Messenger.PostbackButton
title={finishLabel}
payload={JSON.stringify({ type: 'finish', id: todo.id })}
/>
}
/>
))}
</Messenger.GenericTemplate>
</>
);
}
return (
<>
{summary}
{todos.map((todo) => todo.name).join('\n')}
</>
);
};
export default TodoList;

Component Function#

A component is a normal function returns the UI element to be sent. The first argument is an object represent the props of JSX element.

The TodoList can be used like this:

<TodoList todos={[{ id: 1, name: 'foo' }, /* ... */]} />

The TodoList function would then receive a { todos: [/* todos... */] } object while rendering reply. The todos prop are then being used to generate the UIs like:

<>{todos.map(todo => <p>{todo.name}</p>)}</>

As you can see, an array is able to be inserted in element children. This is equal to:

<>
<p>{todo1.name}</p>
<p>{todo2.name}</p>
<p>{todo3.name}</p>
...
</>

Cross-Platform in Component#

The second argument of the component function is an object containing infomations like the current platform. To make your component cross-platform, you can check the platform value and return the corresponded native UI.

const TodoList = ({ todos }: TodoListProps, { platform }) => {
// ...
if (platform === 'messenger') {
return (
// messenger UI element
);
}
// ...

At the end of the function, we usually return a default general UI to support any new platform added in the future.

// ...
return <>{todos.map((todo) => todo.name).join('\n')}</>;
};

Wrapping Children#

One common technic for building component is taking children elements and wrap around it. Let's try it by creating a WithRootMenu component to reuse the list button:

  • Messenger
  • Telegram
  • LINE

Create a new src/components/WithRootMenu.tsx file:

src/components/WithRootMenu.tsx
import Machinat, { MachinatNode } from '@machinat/core';
import { makeContainer } from '@machinat/core/service';
import * as Messenger from '@machinat/messenger/components';
type WithRootMenuProps = {
children: MachinatNode;
todoCount: number;
};
const WithRootMenu = (
{ children, todoCount }: WithRootMenuProps,
{ platform }
) => {
const info = <>You have <b>{todoCount}</b> todos now.</>;
const listLabel = 'Show Todos 📑';
const listData = JSON.stringify({ type: 'list' });
if (platform === 'messenger') {
return (
<>
{children}
<Messenger.ButtonTemplate
buttons={
<Messenger.PostbackButton title={listLabel} payload={listData} />
}
>
{info}
</Messenger.ButtonTemplate>
</>
);
}
return (
<>
{children}
{info}
</>
);
};
export default WithRootMenu;

And change the handleMessage handler like:

src/handlers/handleMessage.tsx
import Machinat from '@machinat/core';
import WithRootMenu from '../components/WithRootMenu';
import { ChatEventContext } from '../types';
const handleMessage = async ({
event,
reply,
}: ChatEventContext & { event: { category: 'message' } }) => {
if (event.type === 'text') {
const matchingAddTodo = event.text.match(/add(\s+todo)?(.*)/i);
if (matchingAddTodo) {
const todoName = matchingAddTodo[2].trim();
return reply(
<WithRootMenu todoCount={3}>
<p>Todo "<b>{todoName}</b>" is added!</p>
</WithRootMenu>
);
}
}
await reply(
<WithRootMenu todoCount={3}>
<p>Hello! I'm a Todo Bot 🤖</p>
</WithRootMenu>
);
};
export default handleMessage;

The WithRootMenu component can be used with additional messages it the element children. Like:

<WithRootMenu todoCount={3}>
<p>Hello! I'm a Todo Bot 🤖</p>
</WithRootMenu>

The element children <p>Hello! I'm a Todo Bot 🤖</p> is available as children prop in component function. We can then return it with the root menu attached below.

Now we are able to build complex, cross-platform and reusable chat UI with the customized Component. Next, we'll show user the real data and provide real features with these UI.
