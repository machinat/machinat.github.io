UI Component

Now we know how to use both general and native UIs. While the UI getting more complex, we might want to reuse them to keep the codes DRY. In this lesson, we will go deeper to make customized chat UI component.

　Time to accomplish: 10 minutes

Building Component#

Let's continue creating the todos list UI. But this time, we'll build the view as a reusable Component module.

First add this type to represent the todo data:

src/types.ts
export type Todo = {
id: number;
name: string;
};

Then follow the guide of your chosen platform to create our first component:

  • Messenger
  • Telegram
  • LINE

Create a src/components/TodoList.tsx file with following content:

src/components/TodoList.tsx
import Machinat from '@machinat/core';
import * as Messenger from '@machinat/messenger/components';
import { Todo } from '../types';
type TodoListProps = {
todos: Todo[];
};
const TodoList = ({ todos }: TodoListProps, { platform }) => {
if (todos.length === 0) {
return <p>You have no todo now.</p>;
}
const summary = <p>You have <b>{todos.length}</b> todos:</p>;
const finishLabel = 'Done ✓';
if (platform === 'messenger') {
return (
<>
{summary}
<Messenger.GenericTemplate>
{todos.slice(0, 10).map((todo) => (
<Messenger.GenericItem
title={todo.name}
buttons={
<Messenger.PostbackButton
title={finishLabel}
payload={JSON.stringify({ type: 'finish', id: todo.id })}
/>
}
/>
))}
</Messenger.GenericTemplate>
</>
);
}
return (
<>
{summary}
{todos.map((todo) => <p>{todo.name}</p>)}
</>
);
};
export default TodoList;

Create a src/components/TodoList.tsx file with following content:

src/components/TodoList.tsx
import Machinat from '@machinat/core';
import * as Line from '@machinat/line/components';
import { Todo } from '../types';
type TodoListProps = {
todos: Todo[];
};
const TodoList = ({ todos }: TodoListProps, { platform }) => {
if (todos.length === 0) {
return <p>You have no todo now.</p>;
}
const summary = <p>You have <b>{todos.length}</b> todos:</p>;
const finishLabel = 'Done ✓';
if (platform === 'line') {
return (
<>
{summary}
<Line.CarouselTemplate
altText={todos.map((todo) => todo.name).join('\n')}
>
{todos.slice(1, 10).map((todo) => (
<Line.CarouselItem
actions={
<Line.PostbackAction
label={finishLabel}
data={JSON.stringify({ type: 'finish', id: todo.id })}
/>
}
>
{todo.name}
</Line.CarouselItem>
))}
</Line.CarouselTemplate>
</>
);
}
return (
<>
{summary}
{todos.map((todo) => <p>{todo.name}</p>)}
</>
);
};
export default TodoList;

The component then can be used in the handlePostback handler like:

Component Function#

A component is a normal function with capitalized name, which returns the UIs to be sent. The component then can be used just like a native component:

<TodoList todos={[{ id: 1, name: 'foo' }, /* ... */]} />

The first argument of component function is an object represent the props of JSX element. Our TodoList function will receive a { todos: [/* ... */] } object while rendering reply message.

We can then use the data from todos prop to return the UI like:

return (
<>
{summary}
{todos.map((todo) => <p>{todo.name}</p>)}
</>
);

Rendering Array#

To render the todos list, we can insert an array of elements in JSX. The {todos.map(todo => <p>{todo.name}</p>)} code above actually works equal to:

<>
...
<p>{todo[1].name}</p>
<p>{todo[2].name}</p>
<p>{todo[3].name}</p>
...
</>

Cross-Platform in Component#

The second argument of the component function is a rendering environments object. To make your component cross-platform, you can check the platform value and return the corresponded native UI.

For example:

const TodoList = ({ todos }: TodoListProps, { platform }) => {
// ...
if (platform === 'messenger') {
return (
// messenger UI element
);
}
// ...

At the end of the function, we can return a default general UI for new platform that added in the future.

// ...
return (
<>
{summary}
{todos.map((todo) => <p>{todo.name}</p>)}
</>
);
};

Wrapping Children#

One another common technic for building component is taking children elements and wrap around it. Let's try it by creating a WithMenu component to reuse the list button.

Create a new src/components/WithMenu.tsx file with the following content:

  • Messenger
  • Telegram
  • LINE
src/components/WithMenu.tsx
import Machinat, { MachinatNode } from '@machinat/core';
import * as Messenger from '@machinat/messenger/components';
type WithMenuProps = {
children: MachinatNode;
todoCount: number;
};
const WithMenu = (
{ children, todoCount }: WithMenuProps,
{ platform }
) => {
const info = <>You have <b>{todoCount}</b> todos now.</>;
const listLabel = 'Show Todos 📑';
const listData = JSON.stringify({ type: 'list' });
if (platform === 'messenger') {
return (
<>
{children}
<Messenger.ButtonTemplate
buttons={
<Messenger.PostbackButton title={listLabel} payload={listData} />
}
>
{info}
</Messenger.ButtonTemplate>
</>
);
}
return (
<>
{children}
{info}
</>
);
};
export default WithMenu;

And change the handleMessage handler like this:

src/handlers/handleMessage.tsx
import Machinat from '@machinat/core';
import WithMenu from '../components/WithMenu';
import { ChatEventContext } from '../types';
const handleMessage = async ({
event,
reply,
}: ChatEventContext & { event: { category: 'message' } }) => {
if (event.type === 'text') {
const matchingAddTodo = event.text.match(/add(\s+todo)?(.*)/i);
if (matchingAddTodo) {
const todoName = matchingAddTodo[2].trim();
return reply(
<WithMenu todoCount={3}>
<p>Todo "<b>{todoName}</b>" is added!</p>
</WithMenu>
);
}
}
await reply(
<WithMenu todoCount={3}>
<p>Hello! I'm a Todo Bot 🤖</p>
</WithMenu>
);
};
export default handleMessage;

Now when your bot saying hello, the menu will be attached like this:

The WithMenu component take additional messages in the element children like:

<WithMenu todoCount={3}>
<p>Hello! I'm a Todo Bot 🤖</p>
</WithMenu>

The element children <p>Hello! I'm a Todo Bot 🤖</p> is then available as children prop in component function. We can simply return it with the menu attached below. Like this:

return (
<>
{children}
<Messenger.ButtonTemplate
buttons={
<Messenger.PostbackButton title={listLabel} payload={listData} />
}
>
{info}
</Messenger.ButtonTemplate>
</>
);

The children prop is useful to decorate messages with some routine attachment. For example: a greeting, a menu, quick replies or asking feedback.

We are able to build complex, cross-platform and reusable chat UI with the customized component now. Next, we'll show user real data with these UIs.
