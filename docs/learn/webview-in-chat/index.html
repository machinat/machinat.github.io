webview-in-chat

Although most chat platforms provide some graphical UI in the chatroom. But web app still performs better on showing large data or doing somplex operations. In this lesson, you will learn how to embed a webview in the chatroom.

To make the UI as easiest as possible, we haven't provide a way to delete todos. A webview is perfect to list all todos and provide a GUI for deleting.

Most chat platforms need special API to display an embedded webview in the chatroom. Follow the guide of the platform you choose to add a button for opening webview:

Add the following codes in the WithRootMenu component:

src/components/WithRootMenu.tsx
import Machinat, { MachinatNode } from '@machinat/core';
import * as Messenger from '@machinat/messenger/components';
const { DOMAIN } = process.env;
type WithRootMenuProps = {
children: MachinatNode;
todoCount: number;
};
const WithRootMenu = (
{ children, todoCount }: WithRootMenuProps,
{ platform }
) => {
const info = <>You have <b>{todoCount}</b> todos now.</>;
const listLabel = 'Show Todos 📑';
const listData = JSON.stringify({ type: 'list' });
const editLabel = 'Edit 📤';
if (platform === 'messenger') {
return (
<>
{children}
<Messenger.ButtonTemplate
buttons={
<>
<Messenger.PostbackButton
title={listLabel}
payload={listData}
/>
<Messenger.UrlButton
messengerExtensions
title={editLabel}
url={`https://${DOMAIN}/webview?platform=messenger`}
/>
</>
}
>
{info}
</Messenger.ButtonTemplate>
</>
);
}
return <>{children}{info}</>;
}
export default WithRootMenu;

Messenger.UrlButton component add a button that opens a web page when user tap it. The messengerExtensions prop need to be set for user login in the webview.

url prop link to the /webview endpoint where our web server is located at. And the platform=messenger querystring tells webview client to adopt messenger platform for login.

LINE server would redirect user to /webview endpoint where our web server is located at. And the platform=line querystring tells webview client to adopt line platform for login.

Webview Platform#

The webview server is hosted by the @machinat/webview platform. Like the chat platforms, your app can receive events from webview client and send the answer events back.

To send the todos data to client, edit the handleWebview file like this:

src/handlers/handleWebview.tsx
import { makeContainer } from '@machinat/core/service';
import TodoController from '../services/TodoController';
import { WebAppEventContext } from '../types';
const handleWebview = makeContainer({
deps: [TodoController],
})(
(todoController) =>
async ({ event, bot, metadata: { auth } }: WebAppEventContext) => {
if (event.type === 'connect') {
const { data } = await todoController.getTodos(auth.channel);
return bot.send(event.channel, {
category: 'webview_push',
type: 'app_data',
payload: { data },
});
}
}
);
export default handleWebview;

After a webview connection is opened and authorized, our app would receive a connect event. It's a good timing to send the todos data to webview client.

event.channel here refer to a connection to the webcview client. We can use the bot.send(connection, event) method to send the push event to client side.

The metadata.auth object contains the authorization infos about the connection. auth.channel here refer to the original chatroom where user comes from. So we can get the todos state data of the chat with TodoController.
